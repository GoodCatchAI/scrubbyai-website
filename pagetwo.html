The problem

AI coding assistants created a paradox.

Productivity up…

  • Junior devs (and AI) ship more code, faster
  • PR volume increases
  • “Looks fine” code slips through because it compiles

…but so is rework

  • Senior engineers spend more time reviewing & debugging
  • Quality gates haven’t evolved for AI-assisted code
  • Rework, churn, and rollbacks climb
The burden transfers upstream: tech leads become full-time reviewers. Scrubby exists to move quality enforcement back to the point of creation.

The solution

A repo-specific quality gate — not a generic linter.

Learns your conventions

Scrubby scans your codebase and discovers the implicit patterns your team already follows.

Blocks “slop” early

Stops inefficient, non-idiomatic code at pre-commit and PR stages — before it becomes review debt.

Not “one-size-fits-all”

Unlike universal linters, Scrubby enforces your team’s patterns automatically.

How it works

A simple flow that shifts quality left — without slowing shipping.

User flow

  • 1) Install extension → setup wizard configures language / API key
  • 2) Scrubby scans codebase → discovers patterns → saves repo-patterns.json
  • 3) Developer writes code → violations appear inline (real-time)
  • 4) On commit → pre-commit checks staged files against patterns
  • 5) On PR → GitHub Actions posts report + blocks merge if needed

What the team gets

  • Less review fatigue (fewer “style/pattern” comments)
  • Fewer regressions (slop blocked before it lands)
  • Faster onboarding (conventions enforced consistently)
  • Clearer PRs (fixes happen earlier, diffs stay clean)

AI technology

Fast static checks + deeper AI analysis — designed to be reliable.

Current (beta)

  • Claude API with an engineered pipeline
  • Stratified sampling for architectural diversity
  • Batch processing with token management
  • Structured prompts → reliable JSON output
  • Dual-layer: fast static checks + AI deep analysis
(We keep the AI honest by making it produce structured, testable outputs.)

Future

  • Custom model training from discovered patterns + feedback
  • Fine-tuned CodeLlama/StarCoder for 10–30× faster inference
  • Local/edge deployment: privacy, speed, and predictable cost
Scrubby is intentionally opinionated: the goal isn’t “universal correctness.” The goal is enforcing your repo’s standards consistently — especially when code is AI-assisted.
